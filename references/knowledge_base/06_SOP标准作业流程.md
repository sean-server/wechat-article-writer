## 一、选题管理

### 1.1 选题来源

```
热点来源（每日必看）：
- 微博热搜（weibo.com/hot搜索）
- 抖音热榜
- 微信指数（微信内搜索"微信指数"）
- 知乎热榜
- 36氪、虎嗅等科技媒体

痛点来源（每周整理）：
- 读者留言/私信
- 社群讨论热点
- 自身经历感悟

固定栏目：
- 周一：科技/趋势
- 周三：职场/成长
- 周五：情感/生活
- 周末：深度/故事
```

### 1.2 选题评估标准

```
评估维度（1-5分）：
1. 目标读者关心程度
2. 话题新鲜度/时效性
3. 素材获取难度
4. 框架匹配度
5. 爆款潜力

选题门槛：综合3分以上才能写
```

## 二、AI生产流程

### 2.1 准备阶段

```
[ ] 确认选题和核心观点
[ ] 从知识库中调取相关素材
[ ] 确定使用框架（参考文案框架库）
[ ] 明确目标字数（2000字左右）
```

### 2.2 生成阶段

```
Step 1：素材收集
↓
输入：选题关键词
输出：3-5个相关素材/案例

Step 2：文案生成
↓
输入：选题 + 素材 + 框架 + 风格要求
输出：完整初稿

Step 3：标题生成
↓
输入：文章主题
输出：20个备选标题

Step 4：人工审核
↓
- 通读检查流畅度
- 调整不合适的表达
- 选择/优化标题
- 敏感词检查
```

### 2.3 发布阶段

```
Step 5：格式处理
↓
添加 front matter：
---
title: 文章标题
author: 萧十一说
coverImage: ./imgs/cover.png
---

Step 6：API发布
↓
推送到公众号草稿箱

Step 7：存档
↓
- 笔记存入 IMA
- 文章存入本地文件夹
- 更新数据追踪表
```

## 三、质量检查清单

### 3.1 发布前必检

```
[ ] 标题吸引人（能让人想点进来）
[ ] 开头有钩子（痛点/故事/悬念/金句）
[ ] 有具体案例或故事（不能全是道理）
[ ] 结构清晰（分点明确，每段有主题句）
[ ] 有情绪共鸣点（让人"扎心"或"有用"）
[ ] 结尾有互动引导（让人想留言/点在看）
[ ] 无敏感词（政治、色情、虚假广告）
[ ] 字数合适（2000字左右，不要太短也不要太长）
[ ] 像和朋友聊天（不要端架子）
```

### 3.2 风格检查

```
✅ 应该有的：
- 口语化表达
- "扎心了"、"绝了"、"说白了"等口头禅
- 具体场景描写
- 真实感受分享

❌ 不应该有的：
- 太官方的表达
- 太说教（"你应该"、"你必须"）
- 太矫情（无病呻吟）
- 空泛道理（没有案例支撑）
```

## 七、常见问题处理

| 问题 | 解决方案 |
|------|---------|
| 不知道写什么 | 看热点榜单，找和职场/情感/成长相关的话题 |
| AI写的太生硬 | 切换到「故事-感悟型」框架，多用第一人称 |
| 不知道用什么框架 | 参考文案框架库的决策树 |
| 标题想不出来 | 先生成20个备选，再从中挑选/组合 |
| 敏感词不确定 | 用词爪等工具检测，或跳过敏感话题 |
