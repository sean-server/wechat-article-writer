# 微信公众号文章创作与发布技能

> 一键创作并发布公众号文章的完整工作流

## 🚀 快速开始（首次使用必读）

### 第一步：运行初始化检查

**首次使用此 skill 时，系统会自动检测运行环境并引导配置。**

运行初始化检查脚本：
```bash
cd ~/.workbuddy/skills/wechat-article-writer
./scripts/init-check.sh
```

该脚本会检查以下内容：
- ✅ **运行环境**：Bun、Python3、curl 是否安装
- ✅ **Python 依赖**：Pillow、requests 等模块
- ✅ **配置文件**：微信公众号凭证、IMA 笔记凭证
- ✅ **知识库文件**：用户画像、写作风格、内容日历等
- ✅ **脚本文件**：所有必需脚本是否完整
- ✅ **网络连接**：API 服务可访问性

### 第二步：根据引导完善配置

如果检查发现有未配置项，脚本会提供交互式引导：
1. **微信公众号凭证**：输入 AppID 和 AppSecret
2. **IMA 笔记凭证**（可选）：输入 ClientID 和 APIKey
3. **知识库完善**：根据模板填写个人信息

### 第三步：开始使用

初始化完成后，直接说：
- `"发布公众号"` - 获取选题建议并创作发布
- `"写篇文章"` - 同上
- `"今天发什么好"` - 仅获取选题建议

## 功能概述

当用户想要发布公众号文章时，此技能会：
1. 根据内容日历给出当天话题类型和选题建议
2. 用户确定选题后，基于知识库自动创作文章（**2000字以上，含内容总结，含参考来源**）
3. **生成封面图**（Unsplash 真实照片）
4. **生成内容配图**（每个主要章节 1 张配图）
5. 使用 **modern 主题**发布到公众号草稿箱
6. 存入 IMA 笔记
7. 输出产出报告

## 前置条件

### 1. 环境安装

#### 1.1 安装 Bun（必需）

Bun 是运行 TypeScript 脚本必需的运行时：

```bash
# macOS/Linux
curl -fsSL https://bun.sh/install | bash

# 安装完成后，重启终端或执行
source ~/.zshrc  # 或 ~/.bashrc

# 验证安装
bun --version
```

#### 1.2 安装 Python3 依赖（必需）

```bash
# 安装 Python 图片处理库
pip3 install Pillow requests urllib3

# 验证安装
python3 -c "import PIL, requests; print('OK')"
```

### 2. 配置文件

凭证文件位置（按优先级自动查找）：

```
# 优先级：skill 自带 config/.env（最优先，可迁移）> ~/.baoyu-skills/.env
config/.env          ← skill 内置，开箱即用
~/.baoyu-skills/.env ← 用户级配置（可选，用于覆盖 skill 配置）
```

#### 2.1 获取微信公众号凭证

1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 左侧菜单 → **开发** → **基本配置**
3. 获取 **AppID** 和 **AppSecret**
4. 填写到 `config/.env`：

```bash
# config/.env
WECHAT_APP_ID=wx你的appid
WECHAT_APP_SECRET=你的appsecret
```

> ⚠️ **重要**：AppSecret 只显示一次，请妥善保存！

#### 2.2 获取 IMA 笔记凭证（可选）

如需自动保存文章到 IMA 笔记：

1. 访问 https://ima.qq.com/agent-interface
2. 获取 **Client ID** 和 **API Key**
3. 填写到 `config/.env`：

```bash
IMA_OPENAPI_CLIENTID=你的clientid
IMA_OPENAPI_APIKEY=你的apikey
```

## 🔧 故障排查

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `bun: command not found` | Bun 未安装 | 按前置条件安装 Bun |
| `ModuleNotFoundError: No module named 'PIL'` | Python 依赖缺失 | `pip3 install Pillow` |
| `获取 access_token 失败` | 微信公众号凭证错误 | 检查 AppID 和 AppSecret |
| `IMA API 调用失败` | IMA 凭证错误或未配置 | 检查凭证或跳过 IMA 保存 |
| 文章没有配图 | 网络问题或 Unsplash 访问失败 | 检查网络，脚本会自动降级处理 |

## 工作流程

### Step 1: 选题建议

当用户说"发布公众号"、"写篇文章"时，执行以下步骤：

1. 读取 `references/knowledge_base/07_内容日历.md` 获取内容日历
2. 根据当天星期几，确定话题类型：
   - 周一：科技/趋势
   - 周二：热点/观点
   - 周三：职场/成长
   - 周四：情感/心理
   - 周五：周末话题
   - 周六/日：深度/故事

3. 从选题备选池中推荐 3-5 个选题，展示给用户确认

展示格式：

```
📅 今日话题：[话题类型]

🔥 推荐选题：
1. [选题1] - [简短说明]
2. [选题2] - [简短说明]
3. [选题3] - [简短说明]

请选择一个选题，或告诉我你想写的主题。
```

用户确认后，进入 Step 2。

### Step 2: 用户确认选题

用户确认后，进入文章创作。

### Step 3: 选择框架

根据选题类型，从 `references/knowledge_base/05_文案框架库.md` 中选择合适框架：

| 选题类型 | 推荐框架 |
|---------|---------|
| 职场/成长 | 痛点-方案型、故事-感悟型 |
| 情感/心理 | 场景-共鸣型、故事-感悟型 |
| 热点/观点 | 悬念-揭秘型、对比反差型 |
| 科技/趋势 | 数据驱动型、递进型 |
| 干货方法 | 清单型、痛点-方案型 |

### Step 4: 文章创作（核心步骤）

#### 4.1 读取知识库
读取以下文件：
- `references/knowledge_base/01_用户画像.md`
- `references/knowledge_base/02_个人自传.md`
- `references/knowledge_base/03_写作风格指南.md`

#### 4.2 章节格式要求（重要）

文章必须使用以下格式，**便于后续自动插入配图**：

```markdown
## 01 [章节标题]

第一部分内容（每段 100-200 字，有具体案例或数据支撑）...

## 02 [章节标题]

第二部分内容...

## 03 [章节标题]

第三部分内容...
```

**章节数量要求**：至少 6 个主要章节（鼓励 7-8 个，深度内容可更多）

#### 4.3 结尾结构要求（重要）

每篇文章**必须包含以下三个结尾模块**，按顺序排列：

**模块一：内容总结（划重点）**

根据文章风格选取以下变体之一，**要点内容必须从文章真实内容中提炼**，不能泛泛而谈：

```
变体A — 干货爽感型（适合方法论/分析类）
---

## 结语

- ✅ [从文章核心观点提炼，20字以内，具体可操作]
- ✅ [从文章第二个核心观点提炼，20字以内，有数据或案例支撑更好]
- ✅ [从文章结论/行动建议提炼，20字以内，让人想收藏]

变体B — 情绪共鸣型（适合观点输出/故事类）
---

## 结语

[金句1]：一句话概括文章最核心的洞察
[金句2]：一句话说明为什么要知道这个
[金句3]：一句话给读者一个行动理由或心理出口

变体C — 清单简洁型（适合清单/盘点类）
---

## 结语

1. [最核心的一点，10-15字]
2. [第二重要的发现，10-15字]
3. [最让人意外的一点，10-15字]
4. [最实用的建议，10-15字]
```

**模块二：互动引导**

根据文章调性从以下变体中选取最自然的一个，**必须与文章主题强相关**：

```
变体A — 好奇追问型（适合科技/观点/分析类）
你觉得呢？

你有没有遇到过类似的情况？或者，你身边有没有这样的例子？
欢迎留言聊聊，**我真的很想知道你的想法**。

变体B — 故事邀请型（适合职场/成长/情感类）
你呢？

有没有哪一点特别戳中你？
有没有你自己的故事想分享？我在留言区等你。

变体C — 轻松调侃型（适合热点/轻松/盘点类）
你呢？有没有哪一条戳中了你？

或者，你还有什么要补充的？
说出来，评论区见。

变体D — 深度思考型（适合深度分析/哲学类）
你呢？

如果AI真的做到了某某事（引用文章中提到的具体场景），你会怎么想？
这个话题值得认真聊聊，**来，说说你的看法**。
```

**模块三：参考来源**

```
---

**参考来源**：

- [来源1标题](真实URL，如 https://www.stats.gov.cn/...)
- [来源2标题](真实URL，如 https://www.zhipin.com/...)
- [来源3标题](真实URL，如 https://www.jiemian.com/...)

*本文参考资料均来自公开渠道，观点仅供参考。*
```

> 参考来源必须使用真实 URL 链接，禁止占位链接。

#### 4.4 字数要求

| 类别 | 最少字数 | 目标字数 |
|------|---------|---------|
| 干货方法类 | 2000 字 | 2500-3000 字 |
| 情感共鸣类 | 1800 字 | 2200-2500 字 |
| 热点评论类 | 1500 字 | 2000-2500 字 |
| 深度分析类 | 2000 字 | 3000-3500 字 |

**字数统计**：正文内容（不含 frontmatter）

#### 4.5 保存文章

将文章保存到 `outputs/` 子目录（相对于本 skill 根目录）：
```
outputs/
└── [YYYY-MM-DD]/
    └── [选题标题]/
        ├── article.md
        └── imgs/
            ├── cover.png
            └── 01-content.png ~ 07-content.png
```

> 可通过 `write_to_file` 工具的 `filePath` 参数指定自定义路径。
> 默认使用 skill 内部路径，便于 skill 整体迁移到其他电脑。

### Step 5: 生成封面图

1. 根据标题关键词判断类别（职场/情感/科技/成长/生活）
2. 从 Unsplash 下载对应类别中最符合文章选题的真实照片作为封面
3. 如果网络失败，使用 PIL 生成图文封面作为备选
4. 自动保存到文章同目录下的 `imgs/cover.png`

执行脚本（两种方式）：
```bash
# 方式1：智能模式（推荐）- 自动保存到 outputs 最新文章目录
python3 scripts/generate_cover.py "文章标题" "类别关键词"

# 方式2：指定路径 - 直接传入完整路径
python3 scripts/generate_cover.py "文章标题" "outputs/2026-03-21/文章标题/imgs/cover.png"
```

> 脚本会自动检测：如果第二个参数是中文类别关键词，自动找到最新的 outputs 目录并保存；如果传入英文路径，直接使用该路径。

### Step 6: 生成内容配图（语义搜索版）

**核心逻辑**：根据每个章节的内容进行语义匹配，而非随机选择。

工作流程：
1. **提取关键词**：分析章节标题 + 首段内容，提取语义关键词
2. **Pexels 语义搜索**：用关键词在 Pexels 搜索最匹配的真实照片（优先）
3. **Unsplash 降级**：Pexels 不可用时，用关键词在词库中语义匹配 → Unsplash 下载
4. **插入文章**：每章节 1 张，自动插入章节标题正下方

执行脚本：
```bash
python3 scripts/generate_content_imgs.py [文章路径]
```

> 脚本会自动处理：关键词提取 → Pexels 搜索 → 降级兜底 → 插入文章

### Step 7: 发布到公众号

1. 读取文章文件（含配图和结尾结构）
2. 添加 front matter 元数据
3. 执行发布脚本（推荐使用 **auto_publish.sh** 自动推荐主题）：

#### 方式一：自动主题推荐（推荐 🎉）

```bash
cd scripts
./auto_publish.sh [文章路径] --author "萧十一说"
```

系统会自动分析文章内容，推荐最合适的主题风格，给您惊喜感！

#### 方式二：手动指定主题

```bash
cd scripts
bun wechat-api.ts [文章路径] --theme <主题> --color <颜色> --author "萧十一说"
```

**主题类型**：
| 主题 | 名称 | 适合类型 |
|------|------|---------|
| `modern` | 现代简洁风 | 干货方法、职场成长 |
| `grace` | 优雅精致风 | 情感故事、深度分析 |
| `simple` | 简约极简风 | 生活休闲、故事感悟 |
| `default` | 经典传统风 | 热点评论、清单盘点 |

**颜色选择**：
| 颜色 | 名称 | 风格 |
|------|------|------|
| `#A93226` | 中国红 | 传统正式 |
| `#0F4C81` | 藏蓝 | 专业稳重 |
| `#009874` | 翡翠绿 | 自然成长 |
| `#D97757` | 暖橙色 | 温暖活力 |
| `#B76E79` | 玫瑰金 | 温柔精致 |
| `#FFB7C5` | 樱花粉 | 甜美少女 |

> 所有脚本均使用 `$(dirname "$0")` 相对定位，无需硬编码路径。

4. 记录返回的 `media_id`

### Step 8: 存入 IMA

使用 skill 内置的 `scripts/ima-api.sh`（无需外部 skill）：

```bash
source scripts/ima-api.sh
result=$(ima_create "文章标题" "框架类型|发布日期|media_id|字数")
doc_id=$(echo "$result" | python3 -c "import sys,json; print(json.load(sys.stdin).get('doc_id',''))")
echo "doc_id: $doc_id"
```

凭证自动从 `config/.env` 加载，无需额外配置。

## 排版规范

### 封面图规范
- 比例：16:9（1280x720）
- 来源：Unsplash 免费照片（按主题分类找最符合文章选题的图片）
- 备用：PIL 生成图文封面
- 底部标注作者名称

### 内容配图规范
- 比例：16:9（900x500）
- 数量：每个章节 1 张（至少 6 张）
- 位置：章节标题正下方
- 来源：**Pexels 语义搜索**（优先）+ Unsplash 降级兜底
- 匹配方式：根据章节内容关键词自动语义搜索，非随机选择

### 文章主题
- **modern**：现代简洁风格（默认推荐）
- **grace**：优雅精致风格
- 可通过 `--theme` 参数指定

## 注意事项

1. **用户确认**：选题必须经过用户确认后才能创作
2. **章节格式**：文章中**必须**使用 `## 01`, `## 02` 等编号格式，确保配图正确插入
3. **结尾三模块**：每篇文章**必须**包含「结语」「互动引导」「参考来源」三个结尾模块
4. **字数门槛**：正文内容不低于 2000 字，不足时需要扩充案例或分析
5. **知识库更新**：每次发布后，可询问用户是否需要更新知识库
6. **数据追踪**：提醒用户在发布后3天/7天/30天更新数据
7. **配图降级**：如果 Unsplash 下载失败，不影响文章发布（跳过内容配图）

## 错误处理

| 错误 | 处理方式 |
|------|---------|
| 封面图下载失败 | 使用 PIL 生成图文封面备选 |
| 内容配图下载失败 | 跳过内容配图，文章正常发布 |
| 字数不足2000字 | 扩充案例或分析，确保达标后再发布 |
| 发布API失败 | 提示用户手动复制内容发布 |
| IMA API失败 | 跳过存入IMA（不影响发布），提示稍后手动保存 |
